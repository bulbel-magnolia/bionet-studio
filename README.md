# bionet-studio

> 一个通用的合成生物网络工作台。当前默认示例面向甘草质量快速评价，将三类质量标志物输入转化为可编辑网络模型，并用前端仿真展示质量指数、荧光读出和采收窗口提示。

## 项目定位

bionet-studio 的核心想法是把生物线路抽象成可编辑的网络：输入层接收环境或实验信号，调控层和响应层整合信号，输出层给出一个或多个读出。平台不绑定某一个应用场景；信号整合、逻辑门、多输出报告器、甘草质量检测等都是可替换的示例项目。

当前 iGEM 展示版本把甘草质量评价作为默认示例。软件将甘草根部提取液中的代表性活性成分抽象为三个输入信号，通过传感器、调控节点、响应节点和报告器组成可编辑网络，再把模拟结果整理为质量指数和红绿灯式判断。

| 输入维度 | 软件中的含义 | 对应项目语境 |
|---|---|---|
| 甘草酸类 | 三萜皂苷相关输入 | 代表甘草主要质量标志物之一 |
| 黄酮类 | 黄酮相关输入 | 反映活性成分积累情况 |
| 酚酸类 | 酚酸相关输入 | 作为质量评价的辅助维度 |

示例参数统一标注为 `illustrative / 待标定`。真实元件进入平台时需要记录来源、标定状态和适用范围。

## 核心功能

| 功能 | 当前实现 |
|---|---|
| 网络搭建 | 从左侧元件库添加传感器、调控器、响应节点、报告器和记忆锁存模块 |
| 参数编辑 | 在右侧检查器调整输入水平、Hill 参数、边权重、阈值和输出增益 |
| 前端仿真 | 使用纯前端仿真内核计算节点活性、输出通道和综合指数 |
| 读出面板 | 支持时间序列、输出通道、决策规则、层活性、剂量响应和真值表 |
| 运行记录 | 保存每次仿真结果，支持恢复、备注、导出和多次运行叠加对比 |
| A/B 对比 | 将两个快照并排比较，查看曲线、终值、峰值和决策差异 |
| 反向设计 | 根据分类、真值表或连续目标搜索权重、输入边强度和阈值候选方案 |
| JSON 导入导出 | 项目可以保存为 JSON 文件，便于复现、分享和版本管理 |
| 中英文界面 | 中文为默认语言，英文界面用于展示和对外交流 |

## 当前示例

| 示例 | 用途 |
|---|---|
| 甘草质量评价示例 | 默认打开的项目示例，展示三类活性成分输入如何汇总为质量指数和采收建议 |
| 信号整合模型 | 中性的三输入网络模板，用于说明平台的通用建模结构 |
| 逻辑 AND 门 | 展示非线性响应节点如何形成双输入逻辑门，并生成真值表 |
| 多输出报告器 | 展示一个网络驱动多个报告通道的读出方式 |

旧的水质检测示例已从当前软件入口移除。相关材料保留为历史参考，不再作为当前项目的默认应用场景。

## 典型使用流程

1. 打开线上预览或本地开发页面。
2. 选择“甘草质量评价示例”，查看默认网络。
3. 在左侧快速设置中调整甘草酸类、黄酮类、酚酸类输入水平。
4. 点击节点或边，在右侧检查器修改参数和连接强度。
5. 在底部读出面板查看质量指数、荧光曲线、剂量响应和采收建议。
6. 点击“运行仿真”保存当前结果，再修改参数进行 A/B 对比。
7. 进入“反向设计”，用目标表生成候选参数方案并一键应用。
8. 使用 JSON 导入导出保存可复现项目。

## 本地运行

首次安装依赖：

```powershell
npm install
```

开发预览：

```powershell
npm run dev
```

生产构建：

```powershell
npm run build
```

提交前检查：

```powershell
npm run verify
```

`npm run verify` 会依次执行类型检查、基础测试、生产构建和静态发布目录自检。构建产物输出到 `docs/`，可直接作为 GitHub Pages 或 iGEM 静态托管目录。

## 核心模型

| 平台对象 | 生物含义 | 当前实现 |
|---|---|---|
| Sensor / Input | 输入信号或感受元件 | Hill 响应，将输入水平转换为活性 |
| Regulator | 中间调控或信号中继 | 加权输入加非线性响应 |
| Response node | 响应层隐藏节点 | 整合多路上游信号 |
| Reporter / Output | 输出通道 | 带表达与成熟动力学的读出 |
| Edge weight | 调控强度或串扰关系 | 正负权重表示激活或抑制 |
| Readout | 用户关心的结果解释 | 曲线、分类、热图、剂量响应、真值表 |
| Reverse target | 目标行为表 | 分类目标、真值表目标或连续数值目标 |

仿真流程：

```text
输入水平
  -> Hill 编码
  -> 节点间加权传递
  -> 非线性响应
  -> 一阶动力学
  -> 输出通道 / 质量指数 / 决策规则
```

反向设计流程：

```text
目标行为表
  -> 生成候选参数
  -> 仿真评估每一行目标
  -> 按损失和命中率排序
  -> 将候选方案应用到当前项目
```

## 数据纪律

1. 示例参数只用于软件演示和模型流程说明，默认标注为 `illustrative / 待标定`。
2. 真实元件进入平台前需要记录 Registry、DOI、论文链接、实验来源或团队标定记录。
3. 甘草质量指数用于快速初筛和采收窗口辅助；正式质量判定继续以药典检测、HPLC 定量和完整质量标准为依据。
4. 真实元件库、参数标定和硬件接口列入后续开发路线，作为下一阶段把原型推进到实验数据联动工具的重点。
