# bionet-studio · 视觉与内容升级方案 v0.1

> 状态：已进入实现阶段。本文档用于统一软件的视觉语言、内容叙事、示例解释和 iGEM 展示口径。主线始终是通用合成生物网络设计与仿真平台；甘草质量检测只是一个应用示例。

---

## 1. 升级目标

本轮升级不是单纯“美化界面”，而是把软件从高保真原型推进成一个更像专业科研工具的产品。

目标分为三层：

1. **视觉层**：建立暗色科学控制台风格，让界面看起来像可长期使用的专业软件，而不是一次性网页 demo。
2. **内容层**：把每个模块的作用讲清楚，让陌生用户和评委知道平台为什么存在、能解决什么问题、输出结果如何解释。
3. **可信层**：持续强调示例值、待标定、来源审计和湿实验验证边界，避免把设计推演误写成真实定量预测。

---

## 2. 视觉语言

推荐命名：**Scientific Command Console**。

中文表述：

> 暗色科学控制台风格：以深色 CAD 画布、分层玻璃面板、信号色节点和仪器式读出为核心，使合成生物网络呈现为可检查、可调参、可复现的设计对象。

英文表述：

> A dark scientific command-center design language, built around a deep CAD canvas, layered glass panels, signal-colored modules, and instrument-like readouts for inspectable and reproducible synthetic biology network design.

视觉关键词：

```text
深色 / 克制 / 精密 / 微光 / 仪器感 / 可解释 / 可复现
```

避免方向：

```text
大面积霓虹 / 卡通图标 / 中药草本主题 / 过度渐变 / 过度动效 / 单一应用绑定视觉
```

甘草示例不单独使用草本或中药风格，以免弱化平台通用性。

## 3. 已实现的视觉升级

当前实现文件：

```text
src/legacy/premium.css
src/main.tsx
```

`premium.css` 作为独立视觉覆盖层加载在 `tokens.css` 和 `components.css` 之后，不改变核心组件结构。

已实现内容：

1. 全局深色空间背景：在 app 背景加入低饱和径向光斑和暗角。
2. 顶栏、侧栏、左右 dock、底部 readout 增加玻璃面板质感。
3. 画布升级为更像 CAD / 控制台的深色工作区。
4. 节点卡片增加内高光、信号色辉光、hover 抬升和选中环。
5. 端口强化为发光 pin，提升连接操作反馈。
6. 读出面板强化为仪器式分析区。
7. A/B compare overlay 强化为旗舰展示模块。
8. 动效统一为短时、克制、可关闭的微交互。

后续可继续增强：

```text
SVG edge glow filter
chart 曲线高光
readout 解释卡
run report 导出样式
```


## 6. 页面级内容升级

### 6.1 Workbench / 工作台

页面目标：让用户完成主闭环。

核心说明：

```text
Build a circuit → tune parameters → inspect outputs → compare runs → export JSON.
```

中文可写：

```text
搭建线路 → 调整参数 → 查看读出 → 对比运行 → 导出项目
```

需要强调：工作台是平台主界面，不属于任何单一应用。

### 6.2 Element Library / 元件库

当前元件库仍是抽象元件面板，不是真实元件数据库。

界面说明建议：

```text
当前元件为抽象模型组件。经核实的真实元件将以带来源、置信度和标定状态的独立条目加入。
```

英文：

```text
Current entries are abstract model components. Verified biological parts will be added with source, confidence, and calibration metadata.
```

### 6.3 Readouts / 读出

读出面板需要让用户知道“这个图怎么读”。

建议每个 readout 都有一句解释：

| Readout | 中文解释 | 英文解释 |
|---|---|---|
| Time series | 显示一个或多个通道随时间变化 | Tracks one or more channels over time |
| Decision rule | 把数值通道映射为离散状态 | Maps a numeric channel to a discrete state |
| Layer activity | 按网络层展示每个节点的稳态活性 | Shows steady-state activity by network layer |
| Dose-response | 扫描一个输入，观察稳态输出 | Sweeps one input and reads the steady-state output |
| Truth table | 对低/高输入组合进行逻辑检查 | Checks low/high input combinations against an output rule |

### 6.4 Runs / 运行记录

运行记录是“可复现性”证据，不只是历史列表。

建议说明：

```text
每次运行记录都会保存项目结构、参数、输入和仿真摘要，可用于复现、对比和导出。
```

英文：

```text
Each run stores the project structure, parameters, inputs, and simulation summary for reproduction, comparison, and export.
```

### 6.5 A/B Compare / 设计对比

A/B 是展示软件价值的核心模块。

建议说明：

```text
保存两个设计状态，比较输出终值、峰值、响应时间、决策状态和参数差异。
```

英文：

```text
Save two design states and compare final output, peak, response time, decision state, and parameter changes.
```

## 7. 示例项目内容升级

当前示例项目建议固定为四类：

```text
Signal integration model    默认通用模板
Logic AND gate              逻辑线路示例
Multi-output reporter       多输出报告器示例
Licorice quality example    应用示例
```

### 7.1 Signal integration model

定位：默认模板。

说明：

```text
一个中性的多输入、多层网络，用于展示平台最基本的建模和读出流程。
```

展示能力：

```text
多输入 Hill 编码
调控节点和响应节点整合
多输出通道
综合 score
时间序列与分类读出
```

### 7.2 Logic AND gate

定位：逻辑线路示例。

说明：

```text
两个输入经过非线性响应节点形成 AND 行为，用于展示平台如何表达基因逻辑。
```

展示能力：

```text
低/高输入组合
真值表
阈值判定
逻辑输出
```

### 7.3 Multi-output reporter

定位：报告器面板示例。

说明：

```text
一个网络同时驱动多个输出通道，用于展示平台处理多报告器读出的能力。
```

展示能力：

```text
多通道输出
通道曲线叠加
综合输出
层活性热图
```

### 7.4 Licorice quality example

定位：应用示例，不是平台主线。

说明：

```text
把三类归一化成分信号整合为一个质量指数，展示平台如何被具体项目复用。
```

展示能力：

```text
领域输入命名
质量指数聚合
质量等级分类
剂量响应
示例项目可导出复现
```

必须强调：

```text
该示例使用 normalized a.u. 输入和 illustrative 参数，不等同于真实甘草质量检测结果。真实应用需要用实验数据标定输入、响应曲线和阈值。
```

## 8. 数据可信边界

所有对外内容必须遵守：

1. 没有来源的参数，只能写成示意值或待标定值。
2. 示例输出是设计推演，不替代湿实验。
3. 真实元件必须有 Registry、DOI、论文链接或实验来源。
4. 用户导入自己的项目时，平台应保存参数来源和标定状态。
5. 甘草示例只能作为应用示例，不能写成平台本体或真实检测结论。

推荐界面短句：

```text
ILLUSTRATIVE DEMO — not calibrated
```

中文：

```text
示意模型，尚未标定
```

## 9. 5 分钟演示脚本

### Step 1：打开默认模板

说明：

```text
这是一个通用信号整合模型。左侧是输入，中间是处理层，右侧是输出和读出。
```

### Step 2：修改输入

操作：调整 Input A / Input B / Input C。

说明：

```text
输入水平经过 Hill 响应转成节点活性，网络会实时更新输出。
```

### Step 3：查看读出

操作：切换 time series、decision rule、layer activity、dose-response。

说明：

```text
同一个模型可以被不同读出解释，适合设计推演、逻辑检查和参数敏感性分析。
```

### Step 4：保存 A/B

操作：保存 Snapshot A，修改参数，保存 Snapshot B。

说明：

```text
平台可以比较两个设计状态，而不是只展示单次结果。
```

### Step 5：导出 JSON

说明：

```text
完整项目可以导出为 JSON，包含拓扑、参数、读出和综合输出设置，方便复现和分享。
```

### Step 6：切换甘草示例

说明：

```text
甘草质量检测只是一个应用示例，用来展示平台如何被具体项目复用。示例参数仍是待标定值。
```

## 11. UI 微文案规则

### 规则 1：按钮短，说明长

按钮应短：

```text
Run
Export
Compare
Add readout
```

说明放 tooltip 或文档中。

### 规则 2：状态词稳定

分类状态不随示例乱变。建议统一三段：

```text
Low / Review / High
```

中文：

```text
偏低 / 待复核 / 较高
```

### 规则 3：不要过度承诺

避免：

```text
predict real quality
detect accurate concentration
validated result
```

推荐：

```text
estimate normalized score
illustrative response
to-be-calibrated threshold
```

### 规则 4：始终保留可复现语言

常用词：

```text
reproducible
exportable
traceable
configurable
inspectable
```

中文：

```text
可复现
可导出
可追溯
可配置
可检查
```
