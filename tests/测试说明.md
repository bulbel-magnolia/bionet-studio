# 测试文件使用说明

本文档用于说明 `tests/sim.basic.test.mjs` 和 `tests/json.import-export.test.mjs` 两个测试文件的用途、运行方式、通过标准和常见失败排查方法。

---

## 1. 测试文件概览

### `tests/sim.basic.test.mjs`

该测试文件用于检查 `src/legacy/sim.js` 中的基础仿真逻辑是否正常。

主要覆盖内容：

- `hill()` Hill 响应函数
- 激活边对输出的增强效果
- 抑制边对输出的降低效果
- `classify()` 阈值分类逻辑
- `crossedTop()` / latch 越阈值逻辑
- `channelFinal()` 最终输出读取
- `channelPeak()` 峰值输出读取
- 仿真结果基础结构是否有效

该测试主要用于确认：

仿真核心函数没有被改坏
节点和边的基础计算逻辑正常
readout 相关基础函数可以正常工作

---

### `tests/json.import-export.test.mjs`

该测试文件用于检查 JSON 导入导出流程是否正常。

主要覆盖内容：

- 导出的 JSON 是否包含正确格式
- 导出后的项目再次导入后是否保持一致
- 是否兼容旧版本 `model` 字段
- 是否兼容直接导入 project 对象
- 旧版本 JSON 缺失 `params` 时是否有 fallback
- 旧版本 JSON 缺失 `readouts` 时是否有 fallback
- 旧版本 JSON 缺失 `aggregate` 时是否有 fallback
- 无效 JSON 项目是否会被拒绝导入

该测试主要用于确认：

项目导出后再次导入不会丢失核心字段
旧版本 JSON 不会导致页面直接失效
节点、边、参数、readouts、aggregate 可以被恢复

---

## 2. 运行环境要求

需要安装 Node.js。

建议使用较新的 Node.js 版本，例如：
 
node -v

确认 Node.js 可以正常运行。

测试文件使用的是 `.mjs` 模块格式，因此需要直接使用 `node` 运行。

---

## 3. 如何运行测试

请在项目根目录执行命令。

### 单独运行仿真基础测试
 
node tests/sim.basic.test.mjs

如果测试通过，应看到类似输出：
 
✓ Hill 响应
✓ 激活边
✓ 抑制边
✓ classification 阈值
✓ crossedTop / latch
✓ 未越过最高阈值
✓ 输出结构
sim basic tests passed

---

### 单独运行 JSON 导入导出测试

 
node tests/json.import-export.test.mjs

如果测试通过，应看到类似输出：
 
✓ 导出结构
✓ 导出后再导入保持一致
✓ 旧版本 model 字段兼容
✓ 直接项目对象兼容
✓ 旧版本缺失字段 fallback
✓ 旧版本字段不完整 fallback
✓ 无效 JSON 项目拒绝导入
json import/export tests passed
 
---

### 一次运行两个测试文件
 
node tests/sim.basic.test.mjs
node tests/json.import-export.test.mjs
 
如果两个测试都通过，说明：
 
仿真核心逻辑正常
JSON 导入导出流程正常

---

## 4. 测试通过标准

### `sim.basic.test.mjs` 通过标准

需要满足：

- `Sim` API 可以从 `src/legacy/sim.js` 正确加载
- `hill()` 返回符合预期的 Hill 响应值
- 增强激活边权重后，输出值应升高
- 使用抑制边时，输出值应降低
- 分类阈值可以正确映射为状态标签
- 输出超过最高阈值时，`crossedTop()` 返回 `true`
- 输出没有超过最高阈值时，`crossedTop()` 返回 `false`
- 仿真结果中可以读取有效的 final 和 peak 数值

---

### `json.import-export.test.mjs` 通过标准

需要满足：

- 导出 JSON 的 `format` 为 `bionet-studio/v2`
- 导出 JSON 中包含完整的 `project`
- 再导入后，`nodes` 保持一致
- 再导入后，`edges` 保持一致
- 再导入后，`params` 保持一致
- 再导入后，`readouts` 保持一致
- 再导入后，`aggregate` 保持一致
- 旧版本 JSON 缺失字段时可以使用 fallback
- 无效 JSON 不会被错误导入

---

## 5. 如何阅读失败信息

两个测试文件都使用了具名测试块。

如果测试失败，会看到类似信息：

Error: [Hill 响应] expected 0 to be close to 0.5
 
其中：
 
[Hill 响应]

表示失败的测试模块。
 
expected 0 to be close to 0.5

表示具体失败原因。

## 6. 常见失败排查

### 失败：`[Hill 响应] expected 0 to be close to 0.5`

说明 `src/legacy/sim.js` 中的 `hill()` 计算逻辑可能被改坏。

应检查：

src/legacy/sim.js

重点检查：
 
Sim.hill

或内部的：

hill()

---

### 失败：`[激活边] activation edge should increase output`

说明激活边权重增强后，输出没有升高。

应检查：
 
src/legacy/sim.js

重点检查：

边权重计算
节点输入汇总
响应节点计算
reporter 输出计算
 
---

### 失败：`[抑制边] inhibitory edge should reduce output`

说明负权重边没有正确产生抑制效果。

应检查：
 
src/legacy/sim.js

重点检查：
 
负权重是否被正确处理
边权重是否参与节点输入计算
节点输入是否被错误截断
 
---

### 失败：`[classification 阈值]`

说明分类规则没有按阈值正确返回标签。

应检查：
 
src/legacy/sim.js
 
重点检查：
 
Sim.classify
 
---

### 失败：`[crossedTop / latch]`

说明最高阈值越过判断异常。

应检查：
 
src/legacy/sim.js

重点检查：
 
Sim.crossedTop
Sim.channelPeak
 
---

### 失败：`[导出后再导入保持一致]`

说明 JSON 导出再导入后，有核心字段丢失或被修改。

应检查：
 
src/legacy/app.jsx

重点检查：

导出 project 时是否完整保留 nodes、edges、params、readouts、aggregate
导入时是否错误覆盖字段
normalizeImportedProject 是否改变了不该改变的字段
 
---

### 失败：`[旧版本缺失字段 fallback]`

说明旧版本 JSON 缺少字段时，没有正确补默认值。

应检查：
 
src/legacy/app.jsx
 
重点检查：

params fallback
readouts fallback
aggregate fallback
channels fallback
meta fallback
 

---

### 失败：`[无效 JSON 项目拒绝导入]`

说明导入逻辑可能接受了无效项目。

应检查导入校验是否包含：
 
Array.isArray(project.nodes)
Array.isArray(project.edges)

无效项目不应该进入页面状态。

---

## 7. 建议的开发流程

每次修改以下文件后，建议运行相关测试。

### 修改仿真逻辑后

如果修改了：
 
src/legacy/sim.js

建议运行：
 
npm test

---

### 修改导入导出逻辑后

如果修改了：
 
src/legacy/app.jsx

尤其是导入、导出、项目 fallback 相关逻辑，建议运行：

 bash
npm test

---

### 修改项目结构或默认模型后

如果修改了：
 
model.js

例如：
 
默认项目字段
readouts 结构
aggregate 结构
channels 结构
节点字段
边字段

建议同时运行：
 
npm test
 
---

## 8. 手动验收 JSON 导入导出

自动测试通过后，还需要进行一次页面手动验收。

步骤：

1. 启动项目页面。
2. 打开一个示例项目。
3. 修改节点、边、参数或 readouts。
4. 点击导出 JSON。
5. 刷新页面。
6. 再次导入刚才导出的 JSON。
7. 检查页面是否恢复一致。

需要确认：

 节点数量一致
节点属性一致
边数量一致
边属性一致
params 一致
readouts 一致
aggregate 一致
页面没有报错
仿真结果可以正常刷新
 

---

## 9. 注意事项

- `sim.basic.test.mjs` 会加载真实的 `src/legacy/sim.js`。
- `json.import-export.test.mjs` 主要验证 JSON 导入导出逻辑和 fallback 规则。
- 如果只是单独验证 JSON 流程，可以只运行 `json.import-export.test.mjs`。
- 如果 `sim.basic.test.mjs` 失败，不一定代表 JSON 测试有问题。
- 如果 JSON 测试通过，也不代表页面交互一定完全正常，仍建议做一次手动导入导出验收。

---
